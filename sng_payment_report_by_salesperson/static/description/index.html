📊 Payment Report by Salesperson

Reporte completo de pagos agrupado por vendedor con análisis detallado de tiempos de cobro.

🎯 Características Principales

  • Vendedor - Nombre del vendedor asignado a cada cliente
  • Cliente - Información completa del pagador
  • Factura - Número y fecha de la factura aplicada
  • Monto sin Impuestos - Base imponible de la factura
  • Monto Pagado - Cantidad recibida del pago
  • Días para Pago - Tiempo transcurrido desde emisión hasta cobro

📈 Vistas Disponibles

  • Vista Lista: Detalle completo de cada pago
  • Vista Pivot: Análisis multidimensional con totales dinámicos
  • Vista Gráfico: Visualización gráfica de los datos

🔍 Filtros Predefinidos

  • Por período: Este Mes, Mes Anterior, Este Año
  • Por velocidad de pago:
    • Pago Rápido (≤ 30 días)
    • Pago Normal (31-60 días)
    • Pago Lento (> 60 días)
  • Por fecha de pago o fecha de factura

📋 Agrupaciones

  • Por Vendedor
  • Por Cliente
  • Por Fecha/Mes de Pago
  • Por Compañía

⚙️ Requisitos

El modelo res.partner debe tener el campo assigned_salesperson_id que vincula cada cliente con su vendedor asignado.

🚀 Acceso

Contabilidad → Reportes → Pagos por Vendedor

📝 Notas

  • Solo muestra pagos confirmados (posted)
  • Solo pagos de entrada (inbound)
  • Solo facturas de cliente (out_invoice, out_refund)
  • Optimizado con vista SQL para mejor rendimiento
